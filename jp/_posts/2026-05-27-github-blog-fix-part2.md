1部で GitHub Pages のデプロイと baseurl の問題を整理したあと、実際の画面をもう一度確認すると、今度は**日本語ページだけ footer の見え方が違う**ことに気づきました。今回の2部では、その原因と修正内容をまとめます。

<br /><br />

![多言語ページネーションと言語切替修正の要約](/assets/img/posts/2026-05-27-github-blog-fix/2026-05-27-part2-pagination-lang-fix-v1.png)

<br /><br />

# 日本語ページだけ見えていた白いボックス
最初に気になったのは、日本語のホームページで footer のすぐ上に**ページネーションの白いボックス**が表示されていたことです。

韓国語ページでは自然に見えるのに、日本語ページでは下の余白が不自然で、footer の位置まで違って見えました。

調べてみると、日本語分岐の pagination 条件が韓国語と同じ動きになっていませんでした。

<br /><br />

# pagination 条件を修正
`_includes/home-feed.html` の日本語分岐では、次の条件を使っていました。

```liquid
{% raw %}
{% if total_posts > 1 %}
{% endraw %}
```

これを次のように修正しました。

```liquid
{% raw %}
{% if total_pages > 1 %}
{% endraw %}
```

単に記事数が2件以上あるかを見るのではなく、**実際にページが2ページ以上あるか**で表示を切り替えるようにしたものです。

<br /><br />

# 1ページあたりの記事数も合わせる
ただし問題はこれだけではありませんでした。日本語側ではページあたりの記事数が別に 5 件へ固定されていました。

```liquid
{% raw %}
{% assign posts_per_page = 5 %}
{% endraw %}
```

このため、韓国語と同じ記事数でも日本語だけ先にページ分割され、ページネーションが出やすくなっていました。

そこで次のように変更しました。

```liquid
{% raw %}
{% assign posts_per_page = site.paginate | default: 20 %}
{% endraw %}
```

これで日本語ページも韓国語と同じ基準になり、不要なページネーション表示が減りました。

<br /><br />

# 言語切り替えのホームURLも修正
最後に確認したのが言語切り替えボタンです。ホーム(`/`) で JP ボタンを押したとき、期待どおり `/jp/` に移動しないケースがありました。

そこで `_includes/_widget/language.html` にホーム専用の分岐を追加し、パスを明示的に分けました。

修正後は次のように動作します。

- `/` → JP をクリック → `/jp/`
- `/jp/` → KR をクリック → `/`

<br /><br />

# まとめ
2部で解決したポイントは3つです。

1. 日本語ページネーション条件を `total_pages > 1` に変更
2. 日本語の `posts_per_page` を韓国語と揃える
3. 言語切り替えでホームURLを正しく分岐させる

この修正で、日本語ページの footer まわりも韓国語ページと同じように自然になりました。

<br /><br />

# 最後に
今回の GitHub ブログ修正は、単なる1つのバグ修正ではなく、

- デプロイの安定化
- パスの修正
- 多言語ページネーションの調整
- 言語切り替えの改善

まで含めた、かなりまとまった作業でした。
