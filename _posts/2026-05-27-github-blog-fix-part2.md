# 일본어 페이지에서만 보이던 흰 박스
가장 먼저 눈에 띈 것은 일본어 홈 페이지의 footer 바로 위에 **페이지네이션 박스가 흰색으로 표시되는 현상**이었습니다.

한국어 페이지에서는 자연스럽게 보이는데, 일본어 페이지에서는 아래쪽 여백이 달라 보였고 전체 footer 위치도 어색해 보였습니다.

원인을 찾아보니 일본어 분기에서 pagination 표시 조건이 한국어와 다르게 동작하고 있었습니다.

<br /><br />

# pagination 조건 수정
`_includes/home-feed.html`의 일본어 분기에서 다음 조건을 수정했습니다.

```liquid
{% raw %}
{% if total_posts > 1 %}
{% endraw %}
```

이 부분을 아래처럼 바꿨습니다.

```liquid
{% raw %}
{% if total_pages > 1 %}
{% endraw %}
```

글 수가 2개 이상인지만 보는 것이 아니라, **실제로 페이지가 2장 이상인지**를 기준으로 보여주게 만든 것입니다.

<br /><br />

# 페이지당 글 수도 한국어와 맞춤
문제는 여기서 끝나지 않았습니다. 일본어 쪽은 페이지당 표시 수가 별도로 5개로 잡혀 있었습니다.

```liquid
{% raw %}
{% assign posts_per_page = 5 %}
{% endraw %}
```

이 값 때문에 한국어와 같은 글 수라도 일본어는 더 빨리 페이지가 나뉘었습니다.

그래서 아래처럼 수정했습니다.

```liquid
{% raw %}
{% assign posts_per_page = site.paginate | default: 20 %}
{% endraw %}
```

이제 일본어도 한국어와 같은 기준을 쓰게 되어 페이지네이션 박스가 불필요하게 뜨는 문제가 줄었습니다.

<br /><br />

# 언어 전환기 홈 경로 수정
마지막으로 언어 버튼도 한 번 더 확인했습니다. 홈(`/`)에서 JP 버튼을 눌렀을 때 경로가 기대와 다르게 동작하는 문제가 있었기 때문입니다.

특히 홈 페이지에서는 일본어 링크가 `/jp/`로 가야 하는데, 기존 분기에서는 조건에 따라 홈 경로가 그대로 유지될 여지가 있었습니다.

그래서 `_includes/_widget/language.html`에서 홈 전용 분기를 추가해 경로를 명확하게 나눴습니다.

이제는 아래처럼 동작합니다.

- `/` → JP 클릭 → `/jp/`
- `/jp/` → KR 클릭 → `/`

<br /><br />

# 정리
2부에서 해결한 핵심은 세 가지입니다.

1. 일본어 페이지네이션 조건을 `total_pages > 1`로 수정
2. 일본어 페이지당 글 수를 한국어 기준과 맞춤
3. 언어 전환기에서 홈 경로를 명확하게 분기 처리

이 작업을 마치고 나니, 일본어 페이지에서도 footer와 레이아웃이 한국어 페이지처럼 자연스럽게 보이게 됐습니다.

<br /><br />
